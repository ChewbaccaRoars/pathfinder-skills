## Knowledge Transfer

- [ ] All code is in the **team's** repo — not yours
- [ ] Code has plain-English comments on every function and key section
- [ ] Champion can explain what the solution does to someone who wasn't involved
- [ ] Champion can make a small modification to the code independently
- [ ] Champion can debug a common error independently
- [ ] Champion knows how to deploy/redeploy changes
- [ ] Champion knows where to get help if stuck (Pathfinder Skills library, your contact info)

## Documentation

- [ ] Self-paced guide is written and published
- [ ] Guide has been reviewed by the champion (they could follow it)
- [ ] Guide is published to the right location:
  - [ ] GitHub (if generic/open-source)
  - [ ] GitLab (if internal/proprietary)
- [ ] Any workshop materials are published alongside the guide

## Solution Handoff

- [ ] Solution is working in production (not just dev/test)
- [ ] All credentials/API keys are in the team's accounts — not yours
- [ ] Triggers are set up under the team's account — not yours
- [ ] Team has appropriate permissions on all files, sheets, and scripts
- [ ] No hardcoded paths, emails, or IDs that reference your personal accounts

## Measurement

- [ ] "Before" metrics were captured at the start of the tour
- [ ] "After" metrics have been measured
- [ ] Impact report is written (hours saved, errors reduced, people impacted)
- [ ] Solution is added to the **Solution Registry**
- [ ] Impact numbers are shared with the team lead

## Retrospective

- [ ] Tour retrospective is written (see 06-tour-retrospective.md)
- [ ] Key learnings are captured
- [ ] Reusable patterns are identified and added to the library
- [ ] Feedback gathered from the team (what worked, what to improve)

## Communication

- [ ] Team lead has been briefed on what was built and next steps
- [ ] Executive sponsor briefed (if applicable)
- [ ] Shareable artifacts posted (Slack, internal wiki, etc.)
- [ ] Champion introduced to the Pathfinder community/network

## Next Tour Prep

- [ ] Next tour team identified
- [ ] Intake form sent to next team
- [ ] Calendar cleared for transition/prep
- [ ] Lessons from this tour applied to the approach for the next one
